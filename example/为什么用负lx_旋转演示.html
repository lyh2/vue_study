🔄 为什么用 -lx？

Three.js 右手系中，绕 Y 轴旋转矩阵的直观解释

📐 右手系绕 Y 轴旋转矩阵（俯视图）
Three.js XZ 平面俯视图（Y 轴朝外）
绕 Y 轴旋转 θ 角（右手系）
标准 3D 旋转矩阵（绕 Y）:
⎡ cosθ   0   sinθ ⎤
⎢   0     1     0   ⎥
⎣ -sinθ   0   cosθ ⎦
展开到 XZ 平面（Y=0）:
x' = x·cosθ + z·sinθ
z' = -x·sinθ + z·cosθ
墙壁只有 X 方向偏移 lx，z=0:
wx = cx + lx·cosθ   (x 分量)
wz = cz + (-lx)·sinθ  (z 分量)
✅ 为什么是 -lx？
因为绕 Y 轴旋转矩阵的第三行第一列是 -sinθ，
代入 x=lx 后，z 分量产生 -lx·sinθ。
代码写的是 wz = cz + -lx * sr * S
📌 为什么是顺时针？
从 Y 正方向往下看（俯视图），右手系的正角度旋转使 X→Z 方向，
在 XZ 平面上表现为顺时针（CW）。
这正是矩阵第三行出现 -sinθ 的原因。

❌ 没加负号（错误） wz = cz + lx·sinθ

单元格中心
绕 Y 旋转矩阵（俯视 CW）:
x' = x·cosθ + z·sinθ
z' = -x·sinθ + z·cosθ
代入 (lx, 0):
wx = cx + lx·cosθ
wz = cz + (-lx)·sinθ
根本原因： Three.js 使用右手坐标系，绕 Y 轴旋转的正角度遵循右手定则。 从 Y 正方向（上方）俯视 XZ 平面，正角度旋转是顺时针方向的， 旋转矩阵为 x' = x·cosθ + z·sinθ， z' = -x·sinθ + z·cosθ。 墙壁只有 X 方向偏移 lx，代入后 z 分量中的 -lx·sinθ 就是代码中 + -lx * sr * S 的来源。
