🔄 为什么用 -lx？

从标准 XY 到 Three.js XZ：逆时针如何变成顺时针

📐 坐标系对比：标准 XY vs Three.js XZ
📐 标准 XY 坐标系（Y ↑）→ CCW
🔄 翻转 Y 轴 → Y 向下（镜像）
📐 Three.js XZ 平面（Z ↓）→ CW
旋转矩阵推导
标准 CCW (XY 坐标系):
x' = x·cosθ - y·sinθ
y' = x·sinθ + y·cosθ
📌 Y 取反 (y' = -y) 代入:
x' = x·cosθ + y·sinθ
z' = -x·sinθ + y·cosθ   (z = -y)
代入 (lx, 0):
wx = cx + lx·cosθ
wz = cz + (-lx)·sinθ
✅ 关键: Y 取反后,
"- y·sinθ" → "+ y·sinθ" (X分量)
"- x·sinθ" 出现在 Z 分量!
🏎️ 墙壁旋转对比
✅ 用了 -lx（正确） wz = cz - lx·sinθ
❌ 没加负号（错误） wz = cz + lx·sinθ
45° (0° ~ 360°)
左墙 (lx = -WALL_X)
右墙 (lx = +WALL_X)
单元格范围
单元格中心
顺时针旋转矩阵:
x' = x·cosθ + z·sinθ
z' = -x·sinθ + z·cosθ
代入 (lx, 0):
wx = cx + lx·cosθ
wz = cz + (-lx)·sinθ
核心原理： 标准 2D 旋转矩阵使用 Y ↑ 坐标系（逆时针 CCW）。 Three.js 俯视图用 XZ 平面（Z ↓），相当于把 Y 轴取反。 对 Y 取反后代入 CCW 矩阵，自然就得到了顺时针 (CW) 形式的矩阵： X 分量的 - y·sinθ 变号成 + z·sinθ， Y 分量的 + x·sinθ 变号成 - x·sinθ 移到 Z 分量。 所以墙壁的 Z 坐标要用 -lx·sinθ，代码写的是 + -lx * sr * S。
