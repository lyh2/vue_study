🔄 为什么用 -lx？

Three.js XZ 平面中的顺时针旋转矩阵可视化

✅ 用了 -lx（正确） wx = cx + lx·cosθ   |   wz = cz - lx·sinθ
❌ 没加负号（错误） wx = cx + lx·cosθ   |   wz = cz + lx·sinθ
45° (0° ~ 360°)
左墙 (lx = -WALL_X)
右墙 (lx = +WALL_X)
单元格范围
单元格中心 (cx, cz)
旋转矩阵（顺时针）:
x' = x·cosθ + z·sinθ
z' = -x·sinθ + z·cosθ
代入 (lx, 0):
wx = cx + lx·cosθ
wz = cz + (-lx)·sinθ
→ 所以 wz 中要用 -lx
为什么？ Three.js 的 XZ 平面俯视图下，Z 轴指向屏幕下方，导致标准的「逆时针旋转矩阵」变成了 顺时针形式： X 分量中 + z·sinθ，Z 分量中 - x·sinθ。 墙壁只在 X 方向有偏移 lx（Z 方向为 0），代入后 Z 分量变为 -lx·sinθ，所以代码里写的是 + -lx * sr * S。
对比两边的右墙方向：正确版本随单元格一起旋转（朝向赛道外侧），错误版本旋转方向反了（朝向赛道内侧）。
