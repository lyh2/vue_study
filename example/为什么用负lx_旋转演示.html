🔄 为什么用 -lx？

2D 旋转矩阵 → 3D 绕 Y 轴旋转矩阵 → 墙壁公式

📐 并行推导：2D 标准 CCW vs 3D 绕 Y 轴 CW
📐 2D 标准旋转矩阵（XY 平面 · CCW）
标准 2D 逆时针 (CCW) 矩阵:
x' = x·cosθ − y·sinθ
y' = x·sinθ + y·cosθ
Y 朝上，正角度 → 逆时针（数学标准）
🧊 3D 绕 Y 轴旋转（2D XZ 俯视图 · CW）
3D 绕 Y 轴顺时针 (CW) 版本:
x' = x·cosθ + z·sinθ
z' = −x·sinθ + z·cosθ
Z 朝下，正角度 → 顺时针（Three.js 俯视）
🧊 3D 透视：绕 Y 轴旋转
完整 3D 绕 Y 轴旋转矩阵:
⎡ cosθ   0   sinθ ⎤
⎢   0     1     0   ⎥
⎣ -sinθ   0   cosθ ⎦
Y 坐标不变，XZ 平面按 CW 旋转
🔍 四步推导：从 2D 矩阵到 -lx
四步推导过程
Step 1：标准 2D 逆时针 (CCW) 旋转矩阵
x' = x·cosθ − y·sinθ
y' = x·sinθ + y·cosθ
这是数学上的标准形式，Y 轴朝上，θ 增加 → 逆时针
Step 2：从 2D XY 扩展到 3D 绕 Y 轴旋转
绕 Y 轴旋转 → XZ 平面旋转，Y 不变
用 z 替换 y，y' = y：
x' = x·cosθ − z·sinθ  (CCW 3D)
y' = y
z' = x·sinθ + z·cosθ  (CCW 3D)
Step 3：俯视方向反转 → 顺时针 (CW)
从 Y+ 俯视 XZ 平面，右手系 +θ 表现为顺时针
所以符号反转，得到 Three.js 实际使用的矩阵：
x' = x·cosθ + z·sinθ  (CW)
y' = y
z' = −x·sinθ + z·cosθ  (CW)
Step 4：代入墙壁坐标 (lx, 0, 0)
墙壁点：x = lx, z = 0 (只有左右偏移，没有前后偏移)
wx = cx + lx·cosθ + 0·sinθ = cx + lx·cosθ  ✅
wz = cz + −lx·sinθ + 0·cosθ = cz + (-lx)·sinθ
🎯 结论：代码中的 wz = cz + -lx * sr * S
负号来自 CW 旋转矩阵第三行第一列的 −sinθ
🏎️ 墙壁旋转对比
✅ 用了 -lx（正确） wz = cz - lx·sinθ
❌ 没加负号（错误） wz = cz + lx·sinθ
45° (0° ~ 360°)
左墙 (lx = -WALL_X)
右墙 (lx = +WALL_X)
单元格范围
单元格中心
CW 旋转矩阵（俯视 XZ）:
x' = x·cosθ + z·sinθ
z' = -x·sinθ + z·cosθ
代入 (lx, 0):
wx = cx + lx·cosθ
wz = cz + (-lx)·sinθ
推导总结：
2D CCW 矩阵 → x' = x·cosθ − y·sinθ, y' = x·sinθ + y·cosθ  (Y↑ 逆时针)
3D 绕 Y 轴 CCW → x' = x·cosθ − z·sinθ, z' = x·sinθ + z·cosθ  (标准 3D)
3D 绕 Y 轴 CW（俯视） → x' = x·cosθ + z·sinθ, z' = −x·sinθ + z·cosθ  (Three.js 实际使用)
代入 (lx, 0) → wz = cz + (-lx)·sinθ ← + -lx * sr * S
