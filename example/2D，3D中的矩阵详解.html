1 什么是矩阵？向量 × 矩阵
📐 矩阵 = 一组有组织排列的数字
一个 2×2 矩阵的样子：
⎡ a   b ⎤
⎣ c   d ⎦
📌 向量 × 矩阵的乘法规则（列向量）：
⎡ a   b ⎤ ⎡ x ⎤ ⎡ a·x + b·y ⎤
⎣ c   d ⎦ ⎣ y ⎦ = ⎣ c·x + d·y ⎦
关键：矩阵的 列 决定了变换后的基向量方向。 第一列 = X 基向量变换后的位置，第二列 = Y 基向量变换后的位置。
💡 核心思想： 矩阵乘法 = 线性变换。把矩阵看作一个"函数"，输入一个向量，输出一个新的向量。 旋转、缩放、切变都是线性变换。平移需要升维（齐次坐标）才能用矩阵表示。

• 第一列 (a, c) = X 基向量 (1,0) 变换后的位置
• 第二列 (b, d) = Y 基向量 (0,1) 变换后的位置
• 矩阵 = [新X基, 新Y基]
2 2D 旋转矩阵 —— 从三角函数到矩阵
🎡 2D 逆时针 (CCW) 旋转
45°
⎡ cosθ   -sinθ ⎤
⎣ sinθ   cosθ    ⎦
📐 2D 旋转矩阵推导
旋转角度 θ，原始点 P(x, y)：
极坐标：x = R·cosα, y = R·sinα
旋转后：x' = R·cos(α+θ), y' = R·sin(α+θ)
用和角公式展开：
x' = R·(cosα·cosθ - sinα·sinθ) = x·cosθ - y·sinθ
y' = R·(sinα·cosθ + cosα·sinθ) = x·sinθ + y·cosθ
写成矩阵形式：
x' = cosθ · x − sinθ · y
y' = sinθ · x + cosθ · y
🎯 这就是 GLSL 中 mat2 rotate2d(float _angle) 的公式来源
📐 标准旋转矩阵函数
GLSL 旋转矩阵函数：
mat2 rotate2d(float _angle) {
  return mat2( cos(_angle), -sin(_angle),
                sin(_angle), cos(_angle));
}
这个矩阵被使用时：
vec2 st = vec2(0.5, 0.3);
st = rotate2d(θ) * st; // 绕原点旋转
🔑 理解矩阵的列
旋转矩阵 R(θ) 的列向量：
• 第一列 (cosθ, sinθ) = X 基向量 (1,0) 旋转后的位置
• 第二列 (-sinθ, cosθ) = Y 基向量 (0,1) 旋转后的位置
✅ 当 θ=0: 矩阵 = 单位矩阵，基向量不变
✅ 当 θ=90°: (cos90, sin90)=(0,1), (-sin90, cos90)=(-1,0)
   → X→(0,1), Y→(-1,0)，逆时针旋转 90°
💡 为什么第二列有负号？ 因为 Y 基向量 (0,1) 旋转 θ 后，x 分量会走向负方向。
直观理解：把 Y 轴正方向（朝上）逆时针转一点，箭头会偏左（x 为负）。
🔑 记住口诀： 「cos 在正对角，sin 在对角线，右上角带负号」
即矩阵 [cosθ, -sinθ; sinθ, cosθ]
3 2D 缩放矩阵
📏 2D 缩放矩阵可视化
1.5
0.8
⎡ 1.50   0.00 ⎤
⎣ 0.00   0.80 ⎦
📐 2D 缩放矩阵推导
缩放 Sx, Sy：
x' = Sx · x
y' = Sy · y
矩阵形式：
⎡ Sx   0 ⎤ ⎡ x ⎤ = ⎡ Sx·x ⎤
⎣ 0   Sy ⎦ ⎣ y ⎦   ⎣ Sy·y ⎦
mat2 scale(vec2 s) {
  return mat2(s.x, 0.0,
              0.0, s.y);
}
🎯 缩放矩阵是对角矩阵，对角线上分别放 X 和 Y 的缩放因子
🔍 观察规律： • Sx > 1 → X 方向拉伸
• Sx < 1 → X 方向压缩
• Sx = Sy → 等比缩放（形状不变）
• 负的缩放 → 镜像翻转
4 平移变换与齐次坐标
🚚 2D 平移可视化
0.8
0.5
📐 平移不是线性变换
平移公式：
x' = x + Tx
y' = y + Ty
⚠️ 平移不能直接用 2×2 矩阵表示！
因为 2×2 矩阵做的是线性变换，必须经过原点 (0,0)。
平移 = 线性变换 + 偏移。
解决方法：升维到齐次坐标
齐次坐标：从 2D → 3D 表示
(x, y) → (x, y, 1)   增加一个 w=1 的维度
⎡ 1   0   Tx ⎤ ⎡ x ⎤ = ⎡ x+Tx ⎤
⎢ 0   1   Ty ⎥ ⎢ y ⎥   ⎢ y+Ty ⎥
⎣ 0   0   1   ⎦ ⎣ 1 ⎦   ⎣ 1    ⎦
💡 为什么需要齐次坐标？ 因为平移不是线性变换（线性变换必须过原点）。
增加 w 维度后，所有仿射变换（旋转+缩放+平移）都统一为 3×3 矩阵乘法。
这就是 3D 图形学中使用 4×4 矩阵（mat4）的原因——包含 3D 平移。
5 组合变换 —— 顺序至关重要
🔄 先旋转 vs 先平移（顺序不同，结果不同）
60°
1.2
🎯 组合变换的正确姿势
先平移后旋转：
st = rotate(θ) * (st + translate);
→ 物体偏离中心旋转（公转）
先旋转后平移（正确顺序）：
st = (rotate(θ) * st) + translate;
→ 物体原地旋转（自转）
🏗️ 图形学标准流程（绕自身中心旋转）：
1️⃣ 平移回原点：st -= vec2(0.5);
2️⃣ 旋转：st = rotate(θ) * st;
3️⃣ 平移回去：st += vec2(0.5);
这就是 GLSL 示例中先减0.5再旋转再加0.5的原因！
🔬 矩阵乘法不满足交换律 一般而言 AB ≠ BA。
变换顺序是从右向左读：
M = T × R × S
应用到点 v 上时：v' = M × v = T × (R × (S × v))
即 先缩放 → 再旋转 → 最后平移
6 3D 旋转矩阵 —— 绕 X / Y / Z 轴
🧊 3D 透视：绕 Y 轴旋转（核心问题）
45°
🎯 绕 Y 轴旋转矩阵：
⎡ cosθ   0   sinθ ⎤
⎢   0     1     0   ⎥
⎣ -sinθ   0   cosθ ⎦
x' = x·cosθ + z·sinθ  (CW)
y' = y
z' = -x·sinθ + z·cosθ  (CW)
📐 三种 3D 旋转矩阵对比
绕 X 轴旋转：
⎡ 1   0     0   ⎤
⎢ 0   cosθ   -sinθ ⎥
⎣ 0   sinθ   cosθ ⎦
Y-Z 平面旋转，X 不变
绕 Y 轴旋转：
⎡ cosθ   0   sinθ ⎤
⎢ 0     1   0   ⎥
⎣ -sinθ   0   cosθ ⎦
X-Z 平面旋转，Y 不变
绕 Z 轴旋转（= 2D 旋转）：
⎡ cosθ   -sinθ   0 ⎤
⎢ sinθ   cosθ     0 ⎥
⎣ 0     0     1 ⎦
X-Y 平面旋转，Z 不变（就是2D旋转+Z维度）
🔑 记住规律： 绕哪个轴旋转，那个轴的坐标就不变。
剩下的二维平面使用标准 2D 旋转矩阵的对应行列。
注意：右手系中，绕 Y 轴的第三行第一列是 -sinθ （这就是 -lx 的来源！）
7 实战：墙壁公式中的 -lx 来源
🏗️ 四步推到墙壁公式
Step 1: 标准 2D CCW 矩阵
x' = x·cosθ − y·sinθ
y' = x·sinθ + y·cosθ
Y↑ 逆时针，数学标准
Step 2: 扩展到 3D 绕 Y 轴
用 z 替换 y，y' = y：
x' = x·cosθ − z·sinθ
z' = x·sinθ + z·cosθ
标准 3D CCW 绕 Y
Step 3: 俯视反转 → CW 版本
从 Y+ 俯视 XZ，右手系 +θ = 顺时针
矩阵第三行符号反转：
x' = x·cosθ + z·sinθ
z' = −x·sinθ + z·cosθ
Three.js 实际使用的 CW 矩阵
Step 4: 代入墙壁坐标
墙壁点：x = lx, z = 0：
wx = cx + lx·cosθ  ✅
wz = cz + (−lx)·sinθ  ← 这就是
wz = cz + -lx * sr * S
🎯 一句话总结： -lx 的负号来自 Three.js 俯视 XZ 平面时使用的 顺时针 (CW) 旋转矩阵第三行第一列的 -sinθ。代入 x = lx, z = 0 后，z 分量自然就是 + (-lx)·sinθ。
8 进阶：YUV 颜色空间变换矩阵
🎨 RGB ↔ YUV 矩阵变换
YUV 是模拟视频编码的颜色空间：
• Y = 亮度（人眼最敏感）
• U / V = 色度（人眼不敏感，可压缩）
这就是视频压缩的底层原理——降采样 UV 通道
RGB → YUV 矩阵（3×3）：
⎡ Y ⎤   ⎡ 0.2126   0.7152   0.0722 ⎤ ⎡ R ⎤
⎢ U ⎥ = ⎢ -0.09991   -0.33609   0.43600 ⎥ ⎢ G ⎥
⎣ V ⎦   ⎣ 0.615   -0.5586   -0.05639 ⎦ ⎣ B ⎦
GLSL 中的 YUV ↔ RGB：
mat3 yuv2rgb = mat3(
  1.0, 0.0, 1.13983,
  1.0, -0.39465, -0.58060,
  1.0, 2.03211, 0.0
);
注意：GLSL 的 mat3 是 列主序（column-major）！
构造函数中每个参数的排列是列优先的。
🔬 为什么颜色能用矩阵？ 颜色 (R,G,B) 本质上就是一个 3 维向量。
矩阵乘法就是在三个维度之间做线性组合。
→ 这就是 颜色分级（Color Grading） 的数学基础！
9 📋 矩阵速查表
变换类型 2D 矩阵 (mat2 / mat3) 说明
单位矩阵 [1 0; 0 1] 什么都不变
旋转 (CCW) [cosθ -sinθ; sinθ cosθ] 逆时针旋转 θ
缩放 [Sx 0; 0 Sy] 对角线元素 = 缩放因子
平移 (齐次) [1 0 Tx; 0 1 Ty; 0 0 1] 需要 3×3 齐次坐标
切变 X [1 k; 0 1] k = 切变系数
镜像 X [-1 0; 0 1] 绕 Y 轴镜像
3D 变换 矩阵 (mat4) 说明
绕 X 轴旋转 [1 0 0; 0 cosθ -sinθ; 0 sinθ cosθ] Y-Z 平面
绕 Y 轴旋转 (CW) [cosθ 0 sinθ; 0 1 0; -sinθ 0 cosθ] X-Z 平面 ★
绕 Z 轴旋转 [cosθ -sinθ 0; sinθ cosθ 0; 0 0 1] X-Y 平面
缩放 3D [Sx 0 0; 0 Sy 0; 0 0 Sz] 对角矩阵
齐次平移 3D [1 0 0 Tx; 0 1 0 Ty; 0 0 1 Tz; 0 0 0 1] 4×4 齐次坐标
⭐ 绕 Y 轴旋转是最特殊的： 第三行第一列的 -sinθ 是因为右手系中从 Y+ 俯视 XZ 平面时，正角度是顺时针而非逆时针。
